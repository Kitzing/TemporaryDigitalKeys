Keychain makes it possible to share keys with anyone, right through the phone! No more need for meeting up to exchange keys.

    • Project description

    The project is a part of the course Mobile Computing on Chalmers University of Technology. The goal was to design a new, innovative action on the theme of authentication and access.

    Project scope

    This specific project is focused on physical acces with the approach of Designing for the real world.

    Goals

    Design challenge

    Creating a prototype that allows the users to share digital keys with each other, without it being more complex than lending an actual key.

    The application was created as an aid for researching the UX-possibilities when solving the problem of the constraints that physical keys create.

    Results

    A Kotlin application with a functioning interaction flow. The front-end and back-end isn’t connected due to the time limitation of the project.

    The project was evaluated on five participants that got to test the application in a real life scenario. The results from this where:

    Future development

    Connect the front-end with the back-end to enable the possibility to update keys and save states then the application is shut down

    Add accounts and the possiblity to sign in
    Future reserach on how NFC should be implemented in order to open RFID-receivers

    First iteration

    Focus:

    Ideate on what features would be needed.

    Results:

    • Mutliple keys
    • Activate and deactivate given keys
    • Give away keys
    • Accept keys
    • Open doors through application
    • Constraint on time or number of uses for keys

    Focus:

    Creating a prototype including the features from the brainstorming in a simplistic design.


    Results:

    A low-fidelity prototype created in Figma, the look is shown below.

    Focus:

    Using Jakob Nielsen’s 10 principles for interaction design to get insights on how the application’s interaction could be improved.

    Results:

    • Simple interactions
    • Consistent design
    • The design is astetic and minimalistic
    • Help and documentation is missing
    • Cancel and back actions are missing

    Second iteration

    Focus:

    Creating a higher fidelity prototype including interactions to communicate our ideas to others.
